**Refs:** handoff §5 (ZhuwenCore contracts), §6 (CP-04 acceptance), §7 (one shared gate
vector set through Go + Swift). `00` §2 (I1, I5), §4 (FR-2, FR-3), §9 (data model), §5 (NFR-2).
Depends on CP-03 (`ZhuwenCore`, vendored `ios/Fixtures/`) and CP-01 (`internal/gate`).

**Acceptance (handoff §6):** EventLog / KnownWordModel / FrontierQueue / CoverageGate /
Selector, with: replay test (I5), gate property tests (I1), selector benchmark (NFR-2).
Plus the standing exit criteria (`plans/exit-criteria.md`).

## Invariants in play
- **I1 coverage gate** — Swift `StoryCandidate` gets a private init; only
  `CoverageGate.evaluate()` produces a populated value (mirrors the Go structural rule).
- **I5 every tap teaches** — `EventLog` is append-only; `KnownWordModel` is a *pure
  projection* of the event list (replayable). Replay test: fold events == batch project.
- **No drift between Go and Swift gates** — one canonical vector set generated by the Go
  reference gate (`internal/gatevec`), vendored to `ios/Fixtures/gate-vectors.json`, and
  asserted by both `go test` and `swift test`.

## Design decisions
1. **Shared gate vectors are Go-authored.** `internal/gate` is the reference. New package
   `internal/gatevec` defines deterministic input vectors, runs each through
   `gate.Evaluate`, and serializes inputs + expected outputs to JSON. `genfixtures` writes
   `ios/Fixtures/gate-vectors.json` (same reproducible-artifact pattern as the packs). A Go
   test locks the vendored file to the current generation; the Swift `CoverageGate` runs the
   same vectors and must reproduce every field.
2. **Language-neutral expected values.** Each vector's expectation is `{pass, denomTokens,
   newTokens, coverageBps, newTypeIDs[], codes[]}`. `coverageBps` uses integer math
   (`(10000*covered + denom/2)/denom`) so Go and Swift agree exactly; `codes` are machine
   reason codes (below), sorted-unique. To feed these, `gate.Result` gains non-breaking
   fields `Codes`, `DenomTokens`, `NewTokens` alongside the existing human `Reasons`.
3. **Reason codes** (stable, both impls): `literal_out_of_lexicon`, `type_budget`,
   `token_budget`, `recurrence`, `frontier`, `grammar`, `proper_noun_gloss`.
4. **KnownWordModel is a deterministic fold** over ordered events. P(known) moves by bounded
   multiplicative/asymptotic steps per event kind; state ∈ {unseen, introduced, learning,
   known, mastered} is derived from P(known) + grade history (FR-2.1). Effective known set
   (FR-2.3) = P≥0.8 ∪ frontier words in `learning`. Rules are documented as heuristic v1
   (placement seeding is CP-05; FSRS memory params are CP-07).
5. **Selector NFR-2 path is bitmap-only.** Stories index to `[UInt64]` type bitmaps; the
   gate hot-loop is `story AND NOT known` + popcount + `⊆ frontier`. Exact token coverage
   for surviving candidates uses per-type token weights precomputed at index build (from the
   story body), never in the hot loop. Benchmark asserts 5,000 stories gated < 50 ms.
6. **No new dependencies** (NFR-5): Swift stdlib only (`UInt64.nonzeroBitCount` popcount,
   `JSONDecoder`). Go adds no deps.

## Deliverables
### factory (Go)
- `internal/gate/gate.go` — add `Result.Codes/DenomTokens/NewTokens` (non-breaking).
- `internal/gatevec/` — `gatevec.go` (`Vectors()`, `Build()`, `JSON()`), `gatevec_test.go`.
- `cmd/genfixtures/main.go` + `internal/fixtures` — also emit `gate-vectors.json`.
- `Makefile` `fixtures` target already runs genfixtures (picks up the new file).

### iOS (`ZhuwenCore`)
- `Bitmap.swift` — `[UInt64]` word bitset (`set/test/andNot/popcount/contains`).
- `CoverageGate.swift` — `GrammarDetector` (marker rules), `CoverageGate.evaluate`,
  `StoryCandidate` (private init, I1), reason codes; `Band`, `GateConfig`, `GateResult`.
- `EventLog.swift` — `Event` (ts, kind, wordID?, storyID?, payload), append-only `EventLog`.
- `KnownWordModel.swift` — `WordState`, `LearningState`, pure projection + effective set.
- `LexiconStore.swift` — read-only lexicon (byID/bySimp, known single-char helper).
- `FrontierQueue.swift` — ordering by (HSK, freq rank, char-familiarity bonus) (FR-2.4).
- `Selector.swift` — `LatticeIndex`, `Selector.select` (gate + FR-3.2 scoring, top-N).

### Tests (`ZhuwenCoreTests`)
- `CoverageGateTests.swift` — **shared vectors** (drive `ios/Fixtures/gate-vectors.json`
  through the Swift gate; assert pass/denom/new/coverageBps/newTypeIDs/codes), plus the
  I1 private-init proof and a hand baseline.
- `EventLogTests.swift` — append-only ordering.
- `KnownWordModelTests.swift` — **replay/I5**: fold == batch; event kinds move P(known) and
  state as specified; effective-known-set membership.
- `FrontierQueueTests.swift` — ordering incl. char-familiarity tie-break.
- `SelectorTests.swift` — gate correctness (rejects >98%-miss / non-frontier), scoring
  prefers frontier/SRS payload, novelty filter, and the **NFR-2 5k-story < 50 ms** benchmark.

## Test plan → acceptance map
- Replay test (I5) → `KnownWordModelTests.testProjectionEqualsIncrementalFold` (+ variants).
- Gate property tests (I1) → `CoverageGateTests` shared-vector suite + randomized property
  test (random known/frontier/token streams: Swift gate agrees with re-derived expectations);
  `testOnlyEvaluateProducesCandidate` (private init).
- Selector benchmark (NFR-2) → `SelectorTests.testGates5kStoriesUnder50ms`.
- Cross-impl (handoff §7) → same `gate-vectors.json` asserted by `go test ./internal/gatevec`
  and `swift test`.

## Exit criteria (standing)
- EC-1 READMEs updated (root + `ios/`) with CP-04 status + run/test commands.
- EC-2 `make build` → `bin/` unchanged (factory commands still build; genfixtures now also
  emits the vector file via `make fixtures`).
- EC-3 `make ci` green; `swift test` green; unit + integration (shared vectors) + e2e
  (selector over the real vendored pack lexicon) present.

## Out of scope (later CPs)
Placement/logistic seeding (CP-05); FSRS memory params + due scheduling (CP-07); SwiftData
persistence of the event log and UI wiring (assembled with the app target); topic-interest
weighting beyond a stub.
